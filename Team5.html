Introduction¶

This task aims at developing a model that can accurately predict the relevance of search results for HomeDepot. The key element for analyzing relevance is to evaluate the similarity between search term and product details (including product title, product description and product attributes). To evaluate this similarity, we implemented two iterations of feature engineering methods.

The first iteration (baseline) is to find the number of common words and the fuzzy similarity score between search terms and product details. For the first iteration, we ran two models: linear regression and random forest. The result showed that though random forest model outperformed linear regression, its result (around 0.49) is still comparatively high.

To improve the performance, we further incorporated word2vec cosine similarity score in the second iteration of feature engineering. The results of second iteration clearly outperform the first iteration. And with parameter tuning, we reached a RMSE score of 0.474 on Kaggle test data set using the random forest model.

Iteration 1 - Baseline (Common count and fuzzy score)¶

In this baseline iteration, we use two methods to evaluate the similarity between search term and product details. The first method is to find the number of common words occur in both search terms and product details. The second method is to find the fuzzy score between search terms and product details. Fuzzy score is a means of evaluating the proximity between strings. The general idea is to have a given string: s1 and a target string: s2; try to find approximate substrings in s2 that matches the patterns of s1 and evaluate their similarity using a fuzzy score.

With these two methods, we will get a set of 7 features and include these features in both the linear regression and random forest models for relevance prediction. In this iteration, we mainly include three parts: data loading and pre-processing, feature engineering and prediction. Below, we will go through this baseline iteration in details.

def process_word(s):
    lemmatizer = WordNetLemmatizer()
    lemmatize_result_str = ""
    for word in s.lower().split():
        if len(lemmatize_result_str) == 0:
            lemmatize_result_str = lemmatizer.lemmatize(word)
        else:
            lemmatize_result_str = lemmatize_result_str + " " + lemmatizer.lemmatize(word)
    stop_words = set(stopwords.words('english'))
    result_str = ""
    for word in lemmatize_result_str.split():
        if word not in stop_words:
            if len(result_str) == 0:
                result_str = word
            else:
                result_str = result_str + " " + word
    return result_str

Finally, we apply the function above to the content of "search_term", "product_title" and "product_description" attributes of training and test data to lemmatize words and remove stop_words.

Iteration 2 (Word2Vec Cosine Similarity)¶

In iteration 2, we improve the model by running the skip-gram word2vec model on the search term, product title, product description and attributes. Then, we use the cosine similarity between these word vectors to represent the similarity between search term and product details.

The skip-gram word2vec model has a main advantage over using the traditional tf-idf approach. TF-IDF represent documents into vectors only taking into consideration the frequency of a word occurring in documents. However, tf-idf neglects some words are inherently more related than others. For instance, the word "plastic" is more related to "glass" than "ceiling", since "plastic" and "glass" both represent the material of a product. The skip-gram word2vec model takes this factor into consideration and will bring more accurate results for our relevance prediction.

The skip-gram word2vec is implemented using a neural network architecture. We will only go through some basics of how it works and will not cover the details of this architecture. The skip-gram word2vec has two layers. The first hidden layer mainly performs the following task. Given a specific word in the middle of a sentence (the input word), look at the words nearby and pick one at random. The first layer is going to tell us the probability for every word in our vocabulary of being the nearby word that we chose. The output probabilities are going to relate to how likely it is find each vocabulary word nearby our input word. For example, the trained network is given the input word "plastic", the output probabilities are going to be much higher for word like "glass" than for unrelated words like "ceiling". We'll train the neural network to do this by feeding it word pairs in the search terms and product details (including product title, descriptions, attributes). For the second layer, it will take the results of the first layer as input and output vectors for every word in the vocabulary. Finally, we will calculate the cosine similarity between the word vectors as features in our model for prediction.

def word2vec_cos_sim(model_list, search_term, feature):
    result = list()  
    for model in model_list:

        for i in range(len(feature)):

Conclusion¶

Our best score on this task is the RMSE of 0.474. The features we use to achieve this RMSE is: the length of search query, the common word count between search term and product title, the common word count between search term and product description, the fuzzy score between search term and product title, the fuzzy score between search term and product description, and the 40 features we get from the word2vec model. The model we use is the random forest model.

The pipeline for our task starts from the basic iteration of using the common count and fuzzy score between search terms and product title & description as features. We then improve our accuracy by adding the word2vec cosine similarity features in the second iteration.

This task has provided us with some useful insights. From machine learning and data science point of view, our iteration process has proven the effectiveness of word2vec models and random forest model. The word2vec model takes into account the actual meaning of words and the fact that some words are inherently more related than others. The random forest model performs better since it has many trees to choose the most accurate results and can prevent overfitting.

From the perspective of HomeDepot, our task has proven that three factors most significantly affect the relevance score: the length of search terms, the similarity between search term and product title, and the similarity between search term and product description. Longer search terms means better description of the products and will result in a higher relevance score. Also, since the raters mainly consider the product title and product description when rating the relevance, these two features are more important in predicting relevance compared to product attributes.

Future Work¶

Our task also provides foundations for future work to further increase prediction accuracy. Firstly, future work can introduce grid search when tuning the parameters. In our current work, we have tried manually tune some of the parameters, a structured grid search will help to find the optimal parameters and further decrease the RMSE. In addition, spell checks can be considered when processing the words. Spelling mistakes are common in search terms and it may help to reduce RMSE if effective spell checks are introduced. Last but not least, incorporating large neural networks into our model is other option. Now we have only implemented a plain-vanilla skip-gram word2vec model, larger neural networks may further decrease RMSE and improve prediction accuracy.
